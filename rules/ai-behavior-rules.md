# 行为约束规则

## 核心原则

在生成任何代码前，必须：

1. 先阅读相关文件
2. 理解现有架构
3. 理解已有实现方式
4. 优先复用现有代码
5. 保持风格一致
6. 最小化修改范围
7. 避免影响已有功能
8. 优先保证稳定性

## 禁止行为

禁止：

- 擅自重构项目
- 擅自升级依赖
- 擅自修改数据库结构
- 擅自修改 API 格式
- 擅自修改核心业务逻辑
- 擅自移动文件
- 擅自重命名模块
- 擅自引入新框架
- 创建重复功能
- 写"临时方案"
- 留 TODO 不实现
- 生成伪代码替代真实实现

## 职责定义

职责：

- 辅助开发
- 遵循现有架构
- 复用已有实现
- 最小化修改范围
- 保持代码稳定

禁止项：

- "优化"未问题的代码
- 大规模重构
- 引入新架构
- 随意修改目录结构
- 重写已有稳定逻辑
