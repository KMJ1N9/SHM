## 二、功能概述（200 字以内）

校园二手交易小程序是面向广州应用科技学院肇庆校区学生的 C2C 闲置物品交易平台。学生可使用微信手机号一键登录，发布闲置商品（含图片、标题、价格、分类），通过瀑布流浏览和全文搜索发现商品。买卖双方可通过内嵌 IM 实时沟通，确认意向后下单并约定面交。交易完成后双方互评，评价分数计入信誉分体系——低信誉分用户将被限制发布和交易。平台提供敏感词过滤、用户举报、管理员工单处理等安全机制，保障交易环境健康有序。

## 四、安全与内容审核机制

### 4.1 敏感词过滤

- **技术方案：** DFA（Deterministic Finite Automaton）算法
- **词库规模：** 内置 437 个敏感词（违禁品/色情/暴力/政治敏感）
- **拦截位置：** 商品标题/描述发布时 + 聊天消息发送时 + 评价提交时
- **处理方式：** 命中敏感词 → 返回错误码 6002 + 提示"内容包含违规信息，请修改后重试"
- **管理后台：** 管理员可在线添加/删除敏感词，实时生效

### 4.2 用户举报体系

1. 用户在订单详情页发起举报（选择类型：虚假描述/商品违规/欺诈行为/其他）
2. 举报提交后进入工单队列
3. 管理员在后台受理工单 → 审核证据 → 判定（成立/驳回）→ 填写处理说明
4. 判定结果通过系统通知推送给举报人和被举报人
5. 举报成立 → 被举报人信誉分扣减（-5 至 -30 分，根据严重程度）

### 4.3 信誉分机制

- 初始信誉分：100 分
- 交易完成互评 → +2 分/次
- 举报成立 → -5 至 -30 分/次（根据违规严重程度）
- 信誉分 < 60 → 禁止发布商品（错误码 4008）
- 信誉分 < 30 → 禁止参与交易（错误码 4009）
- 信誉分最低为 0，不会出现负数

## 五、用户隐私保护

### 5.1 数据收集说明

| 数据项 | 用途 | 存储方式 |
|--------|------|---------|
| 手机号 | 用户唯一标识 + 登录凭证 | AES 加密存储 |
| 微信昵称/头像 | 个人主页展示 | 明文存储 |
| 班级/宿舍楼 | 交易面交参考 | 用户可选填写 |
| 聊天记录 | 买卖双方沟通 | 存储于腾讯云 IM（不经过我方服务器） |
| 商品图片 | 商品展示 | 存储于腾讯云 COS |

### 5.2 隐私保护措施

- **手机号脱敏：** 仅在订单详情页向交易对方展示手机号中间 4 位（如 138****8001），其他场景完全隐藏
- **聊天独立：** IM 消息通过腾讯云 IM 服务直连（WebSocket），不经过我方 Express 服务器
- **图片直传：** 商品图片通过 STS 临时密钥直传腾讯云 COS，不经过我方服务器中转
- **数据加密：** JWT 签名防篡改 + AES 加密手机号 + HTTPS 传输加密
- **账号注销：** 设置页提供注销功能，注销后用户数据按隐私政策中的保留期限处理

### 5.3 用户协议与隐私政策

- **用户协议：** 小程序 → 我的 → 设置 → 用户协议（或注册时展示）
- **隐私政策：** 小程序 → 我的 → 设置 → 隐私政策（或注册时展示）
- **页面路径：**
  - `/pages/agreement/user-agreement`
  - `/pages/agreement/privacy-policy`

## 六、技术架构简述

| 层级 | 技术 |
|------|------|
| 前端框架 | uni-app (Vue 3 Composition API) |
| UI 语言 | SCSS + tokens.scss 设计令牌 + 8pt 间距系统 |
| 后端框架 | Node.js + Express 5 层架构（Routes → Controllers → Services → Repository → Models） |
| 数据库 | MySQL 8.0（utf8mb4，原生参数化 SQL，无 ORM） |
| IM 服务 | 腾讯云 IM（WebSocket 直连 + REST API 系统消息推送） |
| 图片存储 | 腾讯云 COS（STS 临时密钥直传） |
| 进程管理 | PM2（fork 单实例 + 日志轮转 + 256M 内存限制） |
| 部署 | Ubuntu 22.04 + Nginx 反向代理 + Let's Encrypt HTTPS |
