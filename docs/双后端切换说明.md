# 双后端切换说明

## 背景

项目存在两套后端实现，共享同一套 API 契约：

| 后端 | 目录 | 技术栈 | 默认端口 | 状态 |
|------|------|--------|:--:|------|
| Node.js | `server/` | Express 5 层架构 + mysql2 | 3000 | 功能完整，生产可用 |
| Java | `server-java/` | Spring Boot 3.2 + MyBatis + Spring Security | 8081 | Phase 3 已完成，开发中 |

## 架构关系

```
uni-app 前端 (miniprogram/)
        │
        │  HTTP REST API  (统一契约: { code, message, data })
        │
        ├─── Node.js 后端 (server/)   端口 3000
        │
        └─── Java 后端 (server-java/) 端口 8081
```

两个后端是**独立的两个进程**，监听不同端口，互不干扰。它们共同遵守：

- 相同端点路径（`POST /api/auth/login`、`GET /api/products` 等 43 个 API）
- 相同响应格式（`{ code: 0, message: "success", data: {} }`）
- 相同错误码体系（30 个 ErrorCode 一一对应）
- 相同 JWT 鉴权方式（Bearer token + 白名单路径 + tokenVersion 校验）

## 切换方式

前端只有一个地方配置后端地址：

**文件：** `miniprogram/src/api/index.js` 第 20 行

```javascript
// Node.js 后端（当前使用）
const BASE_URL = 'http://10.115.248.247:3000/api';

// Java 后端（开发完成后切换）
// const BASE_URL = 'http://10.115.248.247:8081/api';
```

**改一行代码即可完成切换**，无需修改任何其他文件。前端所有 API 调用（6 个 api 模块 + COS 上传）、图片 URL 解析、Token 刷新全部从 `BASE_URL` 派生。

## 同时运行

两个后端可以同时启动：

```bash
# 终端 1 — Node.js
cd server && npm run dev          # → http://localhost:3000

# 终端 2 — Java
cd server-java && mvn spring-boot:run  # → http://localhost:8081
```

互不冲突，因为没有端口重叠。

## 对比验证

开发 Java 后端时，可以用 curl 同时打两个端口对比 JSON 输出：

```bash
# Node.js 输出作为基准
curl -s http://localhost:3000/api/health | jq .

# Java 输出对比
curl -s http://localhost:8081/api/health | jq .
```

确保契约完全一致（Phase 3.6 契约验证即用此方式）。

## 各阶段使用建议

| 阶段 | 建议后端 | 原因 |
|------|---------|------|
| 当前（Phase 3-4） | Node.js（3000） | 功能完整，前端联调稳定 |
| Java Phase 全部完成后 | Java（8081） | 改一行 BASE_URL 即可 |
| Phase 3.6 契约验证 | 两个同时跑 | 同请求→同响应对比 |
| 生产环境 | 选其一部署 | 最终只保留 Node.js 或 Java |
