# 开发环境搭建指南

**版本：** v1.0 | **更新日期：** 2026-06-03

---

## 一、概述

本文档指导你从零搭建"校园二手交易小程序"的完整本地开发环境，包括：

- Express 后端（Node.js + MySQL）
- uni-app 小程序前端（Vue 3）
- 三个外部云服务（微信小程序、腾讯云 COS、腾讯云 IM）

**预计耗时：** 首次搭建约 1-2 小时（含账号注册审核）。

**适用场景：** 首次克隆项目、换开发机器、新成员加入。

## 二、前置条件

### 2.1 硬件要求

| 项目 | 最低配置 |
|------|---------|
| CPU | 双核 |
| 内存 | 8 GB（HBuilder X + 微信开发者工具同时运行较吃内存） |
| 磁盘 | 5 GB 空闲（含 node_modules + 工具安装） |
| 网络 | 稳定外网连接（需访问微信 API、腾讯云服务） |

### 2.2 软件要求

| 软件 | 版本要求 | 用途 |
|------|---------|------|
| **Node.js** | ≥ 18.x（推荐 20 LTS） | Express 后端运行时 |
| **npm** | ≥ 9.x（随 Node.js 自带） | 包管理 |
| **MySQL** | ≥ 8.0 | 数据库（本地安装或 Docker） |
| **HBuilder X** | 最新版 | uni-app 项目开发、编译为微信小程序 |
| **微信开发者工具** | 最新稳定版 | 小程序调试、预览、真机测试 |
| **Git** | ≥ 2.30 | 版本控制 |
| **VS Code**（可选） | 最新版 | 后端代码编辑（HBuilder X 也能写后端，但 VS Code 体验更好） |

### 2.3 账号要求

| 账号 | 用途 | 获取方式 |
|------|------|---------|
| **微信小程序账号** | 获取 AppID/AppSecret，运行小程序 | [mp.weixin.qq.com](https://mp.weixin.qq.com) 注册 |
| **腾讯云账号** | COS 对象存储 + IM 即时通讯 | [cloud.tencent.com](https://cloud.tencent.com) 注册 |
| **GitHub 账号**（可选） | CI/CD、代码托管 | [github.com](https://github.com) |

> **关于微信小程序主体类型：** 个人主体即可注册，但部分能力受限（无法开通微信支付）。个人主体可满足 MVP 阶段所有需求。后续如需支付功能需升级为企业或个体工商户主体。

## 三、外部服务开通

### 3.1 微信小程序

#### 3.1.1 注册小程序账号

1. 访问 [mp.weixin.qq.com](https://mp.weixin.qq.com)，点击"立即注册"→ 选择"小程序"
2. 填写邮箱、密码（邮箱需未被微信公众平台注册过）
3. 激活邮箱 → 选择"个人"主体 → 填写身份证信息 → 管理员微信扫码绑定
4. 注册完成后进入小程序管理后台

#### 3.1.2 获取 AppID 和 AppSecret

1. 登录 [小程序管理后台](https://mp.weixin.qq.com)
2. 左侧菜单：**开发** → **开发管理** → **开发设置**
3. 记录 **AppID（小程序 ID）** — 公开可见
4. 点击 **AppSecret（小程序密钥）** 旁的"重置"按钮 → 管理员微信扫码 → 复制密钥

> ⚠️ AppSecret 只展示一次，请立即保存到 `.env.development` 中。

#### 3.1.3 配置服务器域名（request 合法域名）

后端 API 和云服务需要在小程序后台配置白名单：

1. 左侧菜单：**开发** → **开发管理** → **开发设置** → **服务器域名**
2. 配置以下域名：

| 类型 | 域名 | 说明 |
|------|------|------|
| request 合法域名 | `https://your-domain.com` | 你的 Express API 服务器 |
| request 合法域名 | `https://console.tim.qq.com` | 腾讯云 IM REST API |
| uploadFile 合法域名 | `https://cos.ap-guangzhou.myqcloud.com` | COS 上传（按实际 region 修改） |
| downloadFile 合法域名 | `https://your-cdn.com` | COS CDN 下载域名 |

> **本地开发阶段可跳过此项：** 微信开发者工具中勾选"不校验合法域名"即可绕过。正式上线前必须配置。

#### 3.1.4 开启手机号获取能力

1. 左侧菜单：**开发** → **开发管理** → **接口设置**
2. 找到"手机号快速验证组件"→ 开通

> 微信对个人主体小程序的 `getPhoneNumber` 能力审核较严格。如果首次审核不通过，准备好说明材料（校园服务场景）重新提交。如长期无法通过，可用微信 `wx.login` + `code2Session` 获取 `openid` 作为临时方案——但会失去手机号实名带来的信任保障。
>
> ⚠️ **降级代价：** 采用 `wx.login` + `code2Session` 方案需：(1) `users` 表新增 `openid` 列作为备用标识；(2) 修改隐私政策 §1.1 说明收集微信 OpenID；(3) 用户协议 §2.1 补充 OpenID 注册方式说明；(4) 前端认证流程从 `getPhoneNumber` 一键授权改为 `wx.login` + 手机号手动填写两步。此方案为临时过渡，仍需持续申请 `getPhoneNumber` 能力。

### 3.2 腾讯云 COS（对象存储）

#### 3.2.1 创建存储桶

1. 登录 [腾讯云 COS 控制台](https://console.cloud.tencent.com/cos)
2. 点击"创建存储桶"
3. 配置如下：

| 配置项 | 建议值 | 说明 |
|--------|--------|------|
| 名称 | `shm-1250000000` | 含 APPID 后缀，自动生成 |
| 所属地域 | `ap-guangzhou`（广州） | 离用户近，延迟低 |
| 访问权限 | **私有读写** | 不开放公开访问，通过 STS 临时密钥授权上传 |
| 版本控制 | 关闭 | MVP 阶段不需要 |

#### 3.2.2 获取 API 密钥

1. 访问 [腾讯云 API 密钥管理](https://console.cloud.tencent.com/cam/capi)
2. 点击"新建密钥" → 记录 **SecretId** 和 **SecretKey**

> ⚠️ 建议使用**子账号**密钥而非主账号密钥，遵循最小权限原则。

#### 3.2.3 配置 CORS（跨域）

存储桶 → 安全管理 → 跨域访问 CORS 设置 → 添加规则：

| 配置项 | 值 |
|--------|-----|
| 来源 Origin | `*`（开发阶段）；正式上线改为小程序域名 |
| 操作 Methods | `PUT, POST, GET, HEAD` |
| Allow-Headers | `*` |
| Expose-Headers | `ETag, x-cos-request-id` |

#### 3.2.4 配置 CDN 加速（可选，开发阶段可跳过）

1. 存储桶 → 域名与传输管理 → 自定义 CDN 加速域名
2. 添加域名并完成 CNAME 解析
3. 将 CDN 域名填入 `.env` 的 `COS_CDN_BASE_URL`

---

### 3.3 腾讯云 IM（即时通讯）

#### 3.3.1 创建 IM 应用

1. 登录 [腾讯云 IM 控制台](https://console.cloud.tencent.com/im)
2. 点击"创建应用"→ 填写应用名称（如 "校园二手交易"）
3. 创建完成后进入应用详情

#### 3.3.2 获取 IM 配置信息

在应用详情页记录以下信息：

| 配置项 | 位置 |
|--------|------|
| **SDKAppID** | 应用详情页顶部，格式 `1400xxxxxx` |
| **SecretKey** | 开发辅助工具 → UserSig 生成 → 密钥 |

#### 3.3.3 创建管理员账号（可选，使用默认即可）

IM 控制台 → 账号管理 → 已有默认 admin 账号。如需修改，对应更新 `.env` 中的 `IM_ADMIN_ACCOUNT`。

#### 3.3.4 开通 REST API 调用权限

IM 控制台 → 安全设置 → 开启"REST API 调用"（默认开启）。

> **免费档限制：** 日活跃用户数 ≤ 100。超过后需升级（约 ¥500/月）。MVP 阶段完全够用。

## 四、本地开发环境搭建

### 4.1 克隆项目

```bash
git clone <repo-url>
cd campus-market
```

### 4.2 安装服务端依赖

```bash
cd server
npm install
```

> `mysql2` 是纯 JavaScript 驱动（无原生编译步骤），Windows/Mac/Linux 均可直接安装，无需额外构建工具。

### 4.3 配置环境变量

```bash
# 1. 从模板创建开发环境配置
cp .env.example .env.development

# 2. 用编辑器打开 .env.development，填入真实值
```

需要填入的核心变量（其余可用默认值）：

| 变量 | 填入内容 | 来源 |
|------|---------|------|
| `WX_APPID` | 你的小程序 AppID | 微信小程序后台 → 开发设置 |
| `WX_APPSECRET` | 你的小程序 AppSecret | 微信小程序后台 → 开发设置 |
| `JWT_ACCESS_SECRET` | 随机生成 32 位密钥 | 终端执行：`node -e "console.log(require('crypto').randomBytes(32).toString('hex'))"` |
| `JWT_REFRESH_SECRET` | 随机生成 32 位密钥 | 同上（不要跟 ACCESS_SECRET 相同） |
| `IM_SDK_APP_ID` | 腾讯云 IM 的 SDKAppID | 腾讯云 IM 控制台 |
| `IM_SECRET_KEY` | 腾讯云 IM 的 SecretKey | 腾讯云 IM 控制台 |
| `COS_BUCKET` | COS 存储桶名称（含 APPID） | 腾讯云 COS 控制台 |
| `COS_REGION` | COS 存储桶地域 | 默认 `ap-guangzhou` |
| `COS_SECRET_ID` | 腾讯云 API SecretId | 腾讯云 API 密钥管理 |
| `COS_SECRET_KEY` | 腾讯云 API SecretKey | 腾讯云 API 密钥管理 |
| `DB_HOST` | MySQL 主机地址 | 本地开发默认 `127.0.0.1` |
| `DB_PORT` | MySQL 端口 | 默认 `3306` |
| `DB_USER` | MySQL 用户名 | 默认 `root` |
| `DB_PASSWORD` | MySQL 密码 | 你的 MySQL root 密码 |
| `DB_NAME` | MySQL 数据库名 | 开发环境用 `campus_market_dev` |

开发环境默认值（无需修改，除非特殊需求）：

```bash
NODE_ENV=development
PORT=3000
DB_HOST=127.0.0.1
DB_PORT=3306
DB_USER=root
DB_PASSWORD=
DB_NAME=campus_market_dev
CORS_ORIGIN=*
LOG_LEVEL=debug
RATE_LIMIT_GLOBAL=999          # 开发环境放开限流
RATE_LIMIT_SENSITIVE=999
ENABLE_CRON=false               # 开发环境不跑定时任务
```

### 4.4 初始化数据库

```bash
# 0. 确保 MySQL 已安装并运行，创建开发数据库
mysql -u root -p -e "CREATE DATABASE campus_market_dev CHARACTER SET utf8mb4"

# 1. 在 server/ 目录下执行迁移
npm run db:migrate

# 预期输出：
# ✓ 已应用 4 个迁移
# ✓ 数据库迁移完成

# 2. 插入种子数据（可选）
npm run db:seed

# 预期输出：
# ✓ 分类预设 6 条（预期值，验收时核对实际种子脚本）
# ✓ 测试用户 4 条（预期值，验收时核对实际种子脚本）
# ✓ 示例商品 15 条（预期值，验收时核对实际种子脚本）
# ✓ 种子数据插入完成
```

可以用 MySQL Workbench 或命令行 `mysql -u root -p campus_market_dev -e "SHOW TABLES"` 查看表结构。

### 4.5 启动开发服务器

```bash
# 在 server/ 目录下执行
npm run dev

# 预期输出：
# [2026-06-03T10:00:00.000Z] INFO: Server started on http://localhost:3000
# [2026-06-03T10:00:00.000Z] INFO: Environment: development
```

**验证后端启动成功：**

```bash
curl http://localhost:3000/api/health

# 预期返回：
# { "status": "ok", "timestamp": "2026-06-03T10:00:00Z", "db": "connected" }
```

### 4.6 启动小程序前端

#### 步骤一：HBuilder X 打开项目

1. 启动 HBuilder X
2. 文件 → 打开目录 → 选择 `miniprogram/` 文件夹
3. 等待项目加载完成（首次可能需要下载 uni-app 编译器插件）

#### 步骤二：配置微信小程序 AppID

1. 打开 `miniprogram/manifest.json`
2. 切换到"微信小程序配置"Tab
3. 在"微信小程序 AppID"中填入你的 AppID
4. 保存

#### 步骤三：配置 API 地址

打开 `miniprogram/api/` 下的模块文件（如 `auth.js`），将 `BASE_URL` 改为你的后端地址：

```js
// 方式一：开发环境直连本地（需微信开发者工具开启"不校验合法域名"）
const BASE_URL = 'http://localhost:3000/api';

// 方式二：使用内网穿透工具（如 ngrok）获取公网 HTTPS 地址
const BASE_URL = 'https://xxxx.ngrok.io/api';
```

#### 步骤四：运行到微信开发者工具

1. HBuilder X 菜单：运行 → 运行到小程序模拟器 → 微信开发者工具
2. 首次需要配置微信开发者工具路径：
   - 微信开发者工具 → 设置 → 安全 → 开启"服务端口"
   - HBuilder X 会自动检测并连接
3. 编译成功后，微信开发者工具自动打开并加载小程序

## 五、验证各模块可用

按顺序验证以下功能，确保所有模块正常：

| 序号 | 验证项 | 操作 | 预期结果 |
|:--:|------|------|------|
| 1 | 后端健康检查 | `curl localhost:3000/api/health` | 返回 `{"status":"ok"}` |
| 2 | 数据库读写 | 调用任意 GET 接口（如商品列表） | 返回 200 + 数据（可能为空数组） |
| 3 | 手机号授权注册/登录 | 在微信开发者工具中点击"获取手机号"按钮 | `getPhoneNumber` 获取 code → POST `/api/auth/login` → 服务端返回 JWT，完成注册/登录 |
| 4 | COS 图片上传 | 在发布商品页选择图片 → 提交 | 图片上传到 COS，返回 CDN URL |
| 5 | IM 登录 | 登录后检查 IM SDK 初始化状态 | `TIM.EVENT.SDK_READY` 事件触发 |
| 6 | IM 消息收发 | 用一个账号给另一个账号发消息 | 消息发送成功，对方能收到 |

> **建议：** 第 1-4 项在每轮开发开始前快速过一遍，确保环境没被误改动。

## 六、开发工作流建议

### 6.1 日常开发流程

```
1. git pull（拉取最新代码）
2. npm run dev（启动后端）
3. HBuilder X 运行小程序到开发者工具
4. 编写代码（后端 routes→controllers→services→repository）
5. npm test（跑测试，确保不破坏已有功能）
6. git add → git commit → git push
```

### 6.2 常用命令速查

```bash
# ---- 后端 ----
npm run dev              # 启动开发服务器（nodemon 热重载）
npm start                # 生产模式启动
npm run db:migrate       # 执行数据库迁移（首次或结构变更时）
npm run db:seed          # 填充种子数据

# ---- 测试 ----
npx vitest               # 监听模式（开发时使用）
npx vitest run           # 全量单次运行
npx vitest run -t "订单状态机"  # 按名称过滤

# ---- 代码质量 ----
npx eslint src/          # 代码检查
npx prettier --check .   # 格式检查
```

### 6.3 多设备开发

如果需要在多台设备上开发（如宿舍台式机 + 图书馆笔记本）：

1. **不要**在设备间拷贝 `.env.development` — 每台设备独立创建
2. **不要**拷贝数据库 — 每台设备独立运行 `npm run db:migrate && npm run db:seed`
3. **不要**拷贝 `node_modules/` — 每台设备独立 `npm install`

### 6.4 开发环境 vs 生产环境对照

| 配置项 | 开发环境 | 生产环境 |
|--------|---------|---------|
| `NODE_ENV` | `development` | `production` |
| 配置文件 | `.env.development` | `.env.production` |
| 日志级别 | `debug` | `info` |
| 限流 | 禁用（999） | `60` / `10` |
| 定时任务 | 关闭 | 开启 |
| CORS | `*` | 具体域名 |
| 微信开发者工具 | "不校验合法域名"勾选 | 全部白名单域名配置 |
| COS Bucket | 可复用生产（建议单独测试 Bucket） | 生产专用 |
| IM 应用 | 建议使用独立的测试 IM 应用 | 生产专用 |

## 七、常见问题排查

### 7.1 后端相关

<details>
<summary><b>Q: MySQL 连接失败 "ECONNREFUSED 3306"</b></summary>

**原因：** MySQL 服务未启动或端口号不对。

**解决：**
```bash
# 确认 MySQL 正在运行
mysql -u root -p -e "SELECT 1"
# 或 Windows 服务中启动 MySQL 服务
```
</details>

<details>
<summary><b>Q: 启动时报 "Missing required env: WX_APPID"</b></summary>

**原因：** `.env.development` 未正确创建或缺少必填变量。

**解决：**
```bash
# 确认文件存在
ls -la server/.env.development

# 检查变量是否填写（不能留空或保留模板值）
cat server/.env.development | grep WX_APPID
# 正确：WX_APPID=wx1234567890abcdef
# 错误：WX_APPID=wx________________ （模板占位符未替换）
```
</details>

<details>
<summary><b>Q: 迁移失败 "Unknown database 'campus_market_dev'"</b></summary>

**原因：** 数据库未创建。

**解决：**
```bash
mysql -u root -p -e "CREATE DATABASE campus_market_dev CHARACTER SET utf8mb4"
```
</details>

### 7.2 微信小程序相关

<details>
<summary><b>Q: HBuilder X 无法启动微信开发者工具</b></summary>

**原因：** 微信开发者工具的服务端口未开启。

**解决：**
1. 微信开发者工具 → 设置 → 安全 → 开启"服务端口"
2. 重启微信开发者工具
3. HBuilder X 重新"运行到小程序模拟器"
</details>

<details>
<summary><b>Q: 小程序请求后端接口报 "request:fail"</b></summary>

**可能原因与解决：**

| 原因 | 解决 |
|------|------|
| 请求的是 `http://localhost` | 微信小程序要求 HTTPS，开发阶段勾选"不校验合法域名" |
| 后端未启动 | 确认 `npm run dev` 正在运行，`curl localhost:3000/api/health` |
| 端口不对 | 检查 `manifest.json` 中的 API 地址端口是否与 `.env` 中 `PORT` 一致 |
</details>

<details>
<summary><b>Q: getPhoneNumber 返回错误</b></summary>

**原因：** 个人主体小程序可能未通过手机号能力审核，或者使用开发工具测试时功能受限。

**解决：**
1. 确认已在微信小程序后台开通"手机号快速验证组件"
2. 在真机上测试（开发者工具对 `getPhoneNumber` 支持有限）
3. 如长期无法通过审核，临时用 `wx.login` + `openid` 方案过渡

> ⚠️ **降级代价：** 见 §3.1.4 完整说明。切换后需额外迁移数据库、更新隐私政策与用户协议。
</details>

### 7.3 腾讯云相关

<details>
<summary><b>Q: COS 上传返回 403 (SignatureDoesNotMatch)</b></summary>

**原因：** STS 临时密钥签名计算错误，通常是 `SecretId` 或 `SecretKey` 填错。

**解决：** 核对 `.env.development` 中的 `COS_SECRET_ID` 和 `COS_SECRET_KEY` 是否与腾讯云控制台完全一致（注意尾部空格）。
</details>

<details>
<summary><b>Q: IM SDK 初始化失败 (SDKAppID 无效)</b></summary>

**原因：** `IM_SDK_APP_ID` 填错，或使用了错误类型的账号。

**解决：** 检查填入的是 `SDKAppID`（数字格式，如 `1400123456`），而非应用名称或其他标识。
</details>

## 八、环境重置

如果需要完全重置开发环境到初始状态：

```bash
# 1. 重置 MySQL 数据库
mysql -u root -p -e "DROP DATABASE IF EXISTS campus_market_dev; CREATE DATABASE campus_market_dev CHARACTER SET utf8mb4"
rm -rf server/logs/*

# 2. 删除依赖和锁文件
rm -rf server/node_modules server/package-lock.json

# 3. 重新安装
cd server && npm install

# 4. 重新执行迁移和种子
npm run db:migrate && npm run db:seed

# 5. 确认配置还在（.env.development 不会被删除）
cat .env.development

# 6. 启动
npm run dev
```
