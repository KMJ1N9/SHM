# ADR-0002：选择腾讯云 IM SDK（非自建 WebSocket）

- **日期：** 2026-06-04
- **状态：** 已采纳
- **决策者：** 单人开发

## 背景

校园二手交易需要买卖双方站内沟通功能。选项：自建 WebSocket 服务 vs 接入第三方 IM SDK。开发周期 13 轮迭代、单人开发。

## 决策

**接入腾讯云 IM SDK（免费档 100 日活），通过 Provider 抽象层封装。**

- 小程序端集成腾讯云 IM SDK
- 服务端通过腾讯云 IM REST API 发送系统消息
- 封装 `server/src/services/im/` Provider 抽象层（`interface IIMProvider`），未来可替换为其他 IM 服务
- 系统消息类型：订单状态变更、评价提醒、举报结果通知、信誉分变动

## 考虑的替代方案

| 方案 | 优点 | 缺点 | 为何未选 |
|------|------|------|---------|
| 自建 WebSocket (ws/socket.io) | 完全自主可控，无第三方依赖 | 需自行处理连接管理、心跳、重连、离线消息、消息持久化——开发量巨大 | 13 轮迭代无法覆盖自建 IM 的完整需求 |
| 云开发 · 实时数据推送 | 与小程序生态集成好 | 功能弱于专业 IM，无消息漫游、已读回执等 | 交易场景需要可靠的消息送达保证 |
| 融云 / 环信 | 专业 IM 厂商 | 免费档限制更严，腾讯云 IM 与微信小程序生态更亲 | 腾讯云 IM 免费档 100 日活足够 MVP |

## 影响范围

- `miniprogram/utils/im.ts` — 小程序端 IM SDK 初始化
- `server/src/services/im/` — Provider 抽象层 + 腾讯云 IM 实现
- `server/src/utils/` — IM 消息发送工具
- IM 系统消息 4 种 JSON Schema（技术架构文档 §八）
- 数据库 `failed_system_messages` 表 — IM 消息发送失败重试

## 后果

### 正面
- 开箱即用的消息收发、已读回执、会话管理
- 免费档覆盖 MVP 阶段 100 日活
- Provider 抽象层隔离，未来可替换
- 与微信小程序生态集成好（腾讯系产品）

### 负面
- 依赖第三方服务，腾讯云 IM 故障时聊天功能不可用
- 免费档日活 100 上限，增长后需付费升级
- 用户数据存储在腾讯云，非自有服务器
- SDK 包体积增加小程序加载时间
